## 1. Выбор подхода: CLI vs TUI

### 1.1 Что такое CLI и TUI

**CLI (Command Line Interface)** — пользователь вводит команду, получает ответ, вводит следующую. Линейное взаимодействие. Примеры: `mysql` в режиме командной строки, `psql`, стандартный bash.

**TUI (Terminal User Interface)** — полноэкранный интерфейс прямо в терминале с несколькими панелями, навигацией стрелками, горячими клавишами, диалогами. Примеры: `htop`, `lazygit`, `ranger`, `midnight commander`, `vim`.

### 1.2 Анализ: за и против

#### Простой CLI (`psql`-стиль)

| ✅ Плюсы | ❌ Минусы |
|---|---|
| Быстро реализовать | Нет контекста: пользователь не видит структуру БД |
| Минимум зависимостей | Нужно помнить имена таблиц и столбцов наизусть |
| Легко интегрировать в скрипты | Результаты скроллятся вверх и теряются |
| Привычен опытным разработчикам | Нет автодополнения без дополнительного кода |
| | Каждый запрос — слепой: непонятно, в какой БД находишься |
| | Неудобен новичкам |

#### TUI (полноэкранный интерфейс)

| ✅ Плюсы | ❌ Минусы |
|---|---|
| Всегда виден контекст: текущая БД, таблица, схема | Сложнее реализовать |
| Навигация без запоминания команд | Зависимость от TUI-библиотеки |
| Результаты фиксированы в своей панели, можно скроллить | Не подходит для использования в скриптах |
| Схема таблицы всегда под рукой | Требует поддержки Unicode в терминале |
| История запросов интерактивна | |
| Подходит и новичкам, и опытным пользователям | |
| Тематически органичен для PixelDB (RPG-стиль) | |

### 1.3 Решение

**Выбираем TUI.** Обоснование:

1. PixelDB — корпоративный проект. TUI обеспечивает удобный интерфейс для администраторов баз данных.
2. Целевая аудитория — инженеры и аналитики, которым важна наглядная структура таблиц. Навигатор решает эту проблему.
3. Основной сценарий использования — ручная работа с БД, а не скриптование. TUI идеален для этого.
4. FTXUI — современная C++17-совместимая библиотека, которая хорошо вписывается в стек проекта.

## 2. Обзор библиотеки FTXUI

### 2.1 Что такое FTXUI

[FTXUI](https://github.com/ArthurSonzogni/FTXUI) (Functional Terminal UI) — header-friendly C++17 библиотека для создания TUI. Используется в реальных проектах (например, встроена в `cmake-init`).

**Ключевые особенности:**
- Декларативный подход: интерфейс описывается как дерево компонентов.
- Встроенная поддержка мыши, клавиатуры, Unicode, цветов.
- Кроссплатформенная (Linux, macOS, Windows).
- Нет системных зависимостей (кроме стандартной библиотеки C++).

### 2.2 Основные концепции FTXUI

```
Screen (экран)
└── Component (дерево компонентов)
    ├── Container::Horizontal / Vertical / Tab
    ├── Input (текстовый ввод)
    ├── Menu (список с выбором)
    ├── Renderer (произвольная отрисовка)
    └── Button, Checkbox, Radiobox...

Element (элементы для рендера)
├── text("Hello")
├── vbox({...}), hbox({...})
├── border(), separator()
├── color(Color::Green, ...)
└── flex(), size()
```

### 2.3 Пример минимального TUI на FTXUI

```cpp
#include <ftxui/component/component.hpp>
#include <ftxui/component/screen_interactive.hpp>
using namespace ftxui;

int main() {
    auto screen = ScreenInteractive::Fullscreen();
    std::string query;

    auto input = Input(&query, "Enter SQL...");
    auto renderer = Renderer(input, [&] {
        return vbox({
            text("PixelDB") | bold | color(Color::Yellow),
            separator(),
            input->Render() | border,
        });
    });

    screen.Loop(renderer);
}
```

### 2.4 Установка через CMake (FetchContent)

```cmake
include(FetchContent)
FetchContent_Declare(
    ftxui
    GIT_REPOSITORY https://github.com/ArthurSonzogni/FTXUI
    GIT_TAG        v5.0.0
)
FetchContent_MakeAvailable(ftxui)

target_link_libraries(pixeldb-tui
    PRIVATE ftxui::screen ftxui::dom ftxui::component
    PRIVATE pixeldb  # наша libpixeldb
)
```

## 4. Панель навигатора

### 4.1 Описание

Левая панель — древовидный браузер баз данных и таблиц. Пользователь видит полную структуру хранилища без необходимости вводить какие-либо команды.

### 4.2 Структура дерева

```
DATABASES                      ← заголовок панели
──────────────────
▶ mydb                        ← свёрнутая БД (▶ = collapsed)
▼ testdb                      ← развёрнутая БД (▼ = expanded)
  ├ employees          [3]       ← таблица + количество строк
  ├ departments           [12]
  └ projects          [7]
▷ archive                     ← БД без таблиц (▷ = empty)
```

**Обозначения иконок:**

| Иконок | Значение |
|---|---|
| `▶` | База данных свёрнута |
| `▼` | База данных развёрнута |
| `▷` | База данных пустая |
| `├` | Таблица (не последняя) |
| `└` | Таблица (последняя в списке) |
| `[N]` | Количество строк (подгружается при разворачивании) |

### 4.3 Управление навигатором

| Клавиша | Действие |
|---|---|
| `↑` / `↓` | Перемещение по дереву |
| `Space` | Развернуть / свернуть базу данных |
| `Enter` | Если таблица — показать её схему в Results; если БД — сделать активной |
| `p` | Preview: SELECT * FROM \<table\> LIMIT 10 — автоматически вставить в редактор и выполнить |
| `d` | Drop: спросить подтверждение, затем DROP TABLE / DROP DATABASE |
| `r` | Refresh: обновить список (перезапросить с сервера) |

### 4.4 Контекстное меню (клавиша `m` или `Right Arrow` на элементе)

При нажатии открывается мини-меню поверх навигатора:

```
╔══════════════════╗
║  employees          ║
╠══════════════════╣
║ ▸ Preview        ║   ← SELECT * LIMIT 10
║   Show Schema    ║   ← показать схему в Results
║   Copy Name      ║   ← скопировать имя в буфер
║   Drop Table     ║   ← с подтверждением
╚══════════════════╝
```

Навигация по меню: `↑` / `↓`, выбор: `Enter`, отмена: `Esc`.

### 4.5 Индикатор активной БД

Активная база данных (в которой выполняются запросы) выделяется жирным и звёздочкой:

```
▼ ★ mydb                     ← активная
  ├ employees          [3]
▶   testdb                   ← неактивная
```

Смена активной БД: навести на БД → `Enter`. Подтверждение в хедере.

### 4.6 Загрузка данных навигатора

При подключении клиент выполняет служебные запросы:
- `SHOW DATABASES;` → список БД (если поддерживается сервером, иначе `LIST DATABASES`)
- При разворачивании БД: `SHOW TABLES;` → список таблиц
- При разворачивании таблицы: `SELECT COUNT(*) FROM <table>;` → количество строк

> **Примечание для разработчика:** если сервер не поддерживает `SHOW DATABASES` и `SHOW TABLES` (в MVP ТЗ их нет), навигатор читает структуру директорий `data/kingdoms/` напрямую через отдельный системный вызов **или** эти команды добавляются в парсер сервера как дополнительные.

## 5. Панель SQL-редактора

### 5.1 Описание

Редактор — основное место работы пользователя. Многострочный текстовый ввод с базовой подсветкой синтаксиса и историей запросов.

### 5.2 Возможности редактора

#### Многострочный ввод
Запрос можно писать в несколько строк. `Enter` — новая строка. Выполнение — только по `F5` или `Ctrl+Enter`.

```
SELECT id, name, salary
FROM employees
WHERE salary > 5
  AND active = TRUE;
```

#### Базовая подсветка синтаксиса

FTXUI позволяет рендерить текст посимвольно с цветами. Реализуется простым лексером прямо в UI-слое:

| Категория | Цвет |
|---|---|
| Ключевые слова (`SELECT`, `FROM`, `WHERE`, `AND`, `OR`, `NOT`, `INSERT`, `UPDATE`, `DELETE`, `CREATE`, `DROP`, `TABLE`, `DATABASE`, `VALUES`, `SET`, `INTO`, `NULL`, `TRUE`, `FALSE`) | Пурпурный |
| Строковые литералы (`'текст'`) | Жёлтый |
| Числа (`42`, `3.14`) | Голубой |
| Операторы (`=`, `!=`, `<`, `>`) | Белый / светло-серый |
| Комментарии (`-- комментарий`) | Серый курсив |
| Остальное (идентификаторы) | Белый |

#### Автодополнение (Tab)

При нажатии `Tab` курсор предлагает варианты из контекста:
- Если начато ключевое слово → завершить его.
- Если после `FROM` или `INTO` → предложить имена таблиц из навигатора.
- Если после `WHERE` → предложить имена столбцов текущей таблицы.

Варианты отображаются в маленьком popup под курсором:

```
SELECT * FROM he█
              ┌─────────────┐
              │ employees      │   ← выбор стрелками
              │ health_log  │
              └─────────────┘
```

Выбор: `↑` / `↓`, применить: `Enter`, отмена: `Esc`.

### 5.3 Управление редактором

| Клавиша | Действие |
|---|---|
| `←` `→` `↑` `↓` | Перемещение курсора |
| `Home` / `End` | Начало / конец строки |
| `Ctrl+Home` / `Ctrl+End` | Начало / конец всего запроса |
| `Backspace` / `Delete` | Удаление символа |
| `Ctrl+A` | Выделить всё |
| `Ctrl+C` | Скопировать выделенное (или всё, если нет выделения) |
| `Ctrl+V` | Вставить из буфера |
| `Ctrl+Z` | Отменить последнее действие (undo, 20 шагов) |
| `Ctrl+K` | Очистить редактор |
| `Tab` | Автодополнение |
| `F5` или `Ctrl+Enter` | **Выполнить запрос** |
| `↑` (когда редактор пуст) | Предыдущий запрос из истории |
| `↓` (когда редактор пуст) | Следующий запрос из истории |

### 5.4 Индикатор состояния редактора

В правом верхнем углу рамки редактора отображается:

```
┌─ SQL Editor ──────────────────── Ln 3 : Col 18 │ READY ─┐
```

Состояния (`READY`, `RUNNING...`, `ERROR`, `OK`):

| Состояние | Цвет | Смысл |
|---|---|---|
| `READY` | Серый | Ожидание ввода |
| `RUNNING...` | Жёлтый мигающий | Запрос выполняется |
| `OK` | Зелёный | Последний запрос успешен |
| `ERROR` | Красный | Последний запрос завершился с ошибкой |

## 6. Панель результатов

### 6.1 Описание

Панель результатов отображает вывод последнего выполненного запроса. Содержимое зависит от типа запроса.

### 6.2 Режим таблицы (для SELECT)

```
┌─ Results ─────────────────────────────── 3 rows │ 42ms ─┐
│  id  │ name          │ salary │ active │ bonus      │      │
│──────┼───────────────┼───────┼───────┼────────────│      │
│  1   │ Smith       │  10   │ TRUE  │ 9.8        │  ▲   │
│  2   │ Johnson       │  9    │ TRUE  │ 9.5        │  █   │
│  3   │ Williams         │  8    │ TRUE  │ 8.2        │  ▼   │
└───────────────────────────────────────────────────────────┘
```

**Особенности отображения:**

- Ширина каждого столбца — максимум из (длина заголовка, длина самого длинного значения), но не более 30 символов.
- Если значение не помещается — обрезать и добавить `…`.
- Вертикальный скроллбар справа, если строк больше, чем высота панели.
- Горизонтальный скролл: `Shift+←` / `Shift+→`, если столбцов много.
- NULL отображается как `∅` (серым курсивом).
- TRUE / FALSE — зелёным / красным для наглядности.

**Навигация по таблице:**

| Клавиша | Действие |
|---|---|
| `↑` / `↓` | Прокрутка строк |
| `PgUp` / `PgDn` | Прокрутка страницами |
| `Shift+←` / `Shift+→` | Горизонтальный скролл |
| `e` | Expand row: открыть выбранную строку в детальном виде |
| `c` | Copy row: скопировать строку как CSV |

**Детальный просмотр строки (`e`):**

При нажатии `e` на строке открывается popup с вертикальным отображением всех полей:

```
╔══════════════════════════════════╗
║  EMPLOYEE DETAILS  (row 1 of 3)     ║
╠══════════════════════════════════╣
║  id      │  1                   ║
║  name    │  Smith             ║
║  salary   │  10                  ║
║  active   │  TRUE                ║
║  bonus   │  9.8                 ║
║  notes     │  King of Gondor,     ║
║           │  returned from...   ║
╠══════════════════════════════════╣
║  [←/→] Prev/Next  [Esc] Close  ║
╚══════════════════════════════════╝
```

Это особенно полезно для длинных TEXT-полей, которые в таблице обрезаются.

### 6.3 Режим сообщения (для INSERT / UPDATE / DELETE)

### 6.6 Режим схемы таблицы

Активируется из навигатора ("Show Schema"). Показывает структуру таблицы:

## 7. Статус-бар

Нижняя строка. Всегда видима. Содержит:

```
F1:Help  F2:History  F5:Run  F9:NewDB  F10:Quit  │  NORMAL  │  mydb
```

**Левая часть:** горячие клавиши верхнего уровня (меняются в зависимости от активного режима).

**Центр:** текущий режим (`NORMAL`, `INSERT`, `NAVIGATOR`, `DIALOG`).

**Правая часть:** имя активной базы данных.

**При ошибке соединения** статус-бар становится красным:
```
[DISCONNECTED]  Reconnecting...  │  Press R to retry
```

## 8. Система горячих клавиш

### 8.1 Глобальные клавиши (работают всегда)

| Клавиша | Действие |
|---|---|
| `F1` | Открыть экран помощи |
| `F2` | Открыть историю запросов |
| `F5` | Выполнить запрос из редактора |
| `F9` | Диалог создания новой базы данных |
| `F10` | Выход (с подтверждением, если есть несохранённый запрос) |
| `Ctrl+Q` | Выход немедленно |
| `Tab` | Переключение фокуса: Навигатор ↔ Редактор ↔ Результаты |
| `Esc` | Закрыть диалог / выйти из автодополнения / снять фокус |

### 8.2 В режиме навигатора (фокус на левой панели)

| Клавиша | Действие |
|---|---|
| `↑` / `↓` | Перемещение по дереву |
| `Space` | Свернуть / развернуть |
| `Enter` | Выбрать элемент (сделать активной БД / preview таблицы) |
| `p` | Preview таблицы (SELECT * LIMIT 10) |
| `s` | Показать схему таблицы |
| `d` | Удалить (с подтверждением) |
| `n` | Создать таблицу в текущей БД |
| `r` | Обновить (refresh) |
| `m` / `→` | Контекстное меню |

### 8.3 В режиме редактора (фокус на редакторе)

| Клавиша | Действие |
|---|---|
| `F5` / `Ctrl+Enter` | Выполнить |
| `Tab` | Автодополнение |
| `Ctrl+K` | Очистить редактор |
| `Ctrl+Z` | Undo |
| `Ctrl+A` | Выделить всё |
| `↑` (пустой редактор) | Предыдущий запрос из истории |
| `↓` (пустой редактор) | Следующий запрос из истории |

### 8.4 В режиме результатов (фокус на панели результатов)

| Клавиша | Действие |
|---|---|
| `↑` / `↓` | Прокрутка строк |
| `PgUp` / `PgDn` | Прокрутка страницами |
| `Home` / `End` | Первая / последняя строка |
| `Shift+←` / `Shift+→` | Горизонтальный скролл |
| `e` | Детальный просмотр строки |
| `c` | Скопировать строку как CSV |
| `C` | Скопировать все результаты как CSV |
| `f` | Фильтр результатов (быстрый поиск по отображаемым данным) |

### 8.5 Визуальная шпаргалка горячих клавиш

Статус-бар меняет содержимое в зависимости от активной панели:

**Навигатор активен:**
```
[SPC]Expand  [ENT]Select  [p]Preview  [s]Schema  [d]Drop  [n]New  [r]Refresh
```

**Редактор активен:**
```
[F5]Run  [TAB]Complete  [Ctrl+K]Clear  [Ctrl+Z]Undo  [↑↓]History
```

**Результаты активны:**
```
[↑↓]Scroll  [e]Expand  [c]CopyRow  [C]CopyAll  [f]Filter  [PgUp/Dn]Page
```

История сохраняется в файле `~/.pixeldb/history.json`:

```json
[
  {
    "timestamp": "2024-01-15T14:32:01Z",
    "query": "SELECT * FROM employees WHERE salary > 5;",
    "success": true,
    "duration_ms": 42
  }
]
```

- Максимум 500 записей. При превышении — удалить самые старые.
- Файл создаётся автоматически при первом запуске.

### 12.2 Главный класс App

### 12.3 Класс Highlighter

```cpp
class Highlighter {
public:
    // Принимает строку SQL, возвращает FTXUI Element

    ftxui::Element highlight(const std::string& sql);

private:
    struct Token {

    };
```

### 12.4 Класс Navigator

```cpp
class Navigator {
public:
    explicit Navigator(pixeldb::Connection& conn);

    ftxui::Component render();

    // Callback-и для взаимодействия с App
    std::function<void(const std::string& db)>    onDatabaseSelected;
    std::function<void(const std::string& query)> onPreviewRequested;
    std::function<void(const std::string& db,
                       const std::string& table)>  onSchemaRequested;

    void refresh();  // перезагрузить структуру с сервера

private:
    struct TreeNode {
        std::string name;
        bool        isDatabase;
        bool        expanded;
        int         rowCount;       // -1 = не загружено
        std::vector<TreeNode> children;
    };

    std::vector<TreeNode> tree_;
    int                   selectedIndex_;
    pixeldb::Connection&  conn_;

    void loadDatabases();
    void loadTables(TreeNode& dbNode);
    void loadRowCount(TreeNode& tableNode, const std::string& db);
};
```

### 12.5 Поток данных при выполнении запроса

```
User presses F5
      │
      ▼
App::executeQuery(sql)
      │
      ├─→ history_.add(sql)
      │
      ├─→ editor_.setState(RUNNING)
      │
      ├─→ conn_.execute(sql)   ← блокирующий вызов
      │         │
      │         ▼
      │   pixeldb::Result result
      │
      ├─→ history_.update(result.success, duration)
      │
      ├─→ editor_.setState(result.success ? OK : ERROR)
      │
      └─→ results_.display(result)
               │
               ├─ if rows    → TableView::render(result.columns, result.rows)
               ├─ if affected → MessageView::renderAffected(result.affected)
               └─ if error   → MessageView::renderError(result.message)
```

> **Важно:** вызов `conn_.execute()` блокирует UI. Для MVP это допустимо (запросы выполняются мгновенно на локальных данных). В будущем — вынести в отдельный поток с отображением `RUNNING...` и возможностью отмены (`Ctrl+C`).

## 14. Сборка и зависимости

### 14.1 CMakeLists.txt для TUI

```cmake
cmake_minimum_required(VERSION 3.20)
project(pixeldb_tui VERSION 1.0.0 LANGUAGES CXX)

set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED ON)

# ─── FTXUI ───
include(FetchContent)
FetchContent_Declare(
    ftxui
    GIT_REPOSITORY https://github.com/ArthurSonzogni/FTXUI
    GIT_TAG        v5.0.0
    GIT_SHALLOW    TRUE
)
FetchContent_MakeAvailable(ftxui)

# ─── Исходники TUI ───
file(GLOB_RECURSE TUI_SOURCES
    app/*.cpp
    screens/*.cpp
    panels/*.cpp
    dialogs/*.cpp
    components/*.cpp
    logic/*.cpp
    utils/*.cpp
)

# ─── Исполняемый файл ───
add_executable(pixeldb-tui main.cpp ${TUI_SOURCES})

target_include_directories(pixeldb-tui PRIVATE .)

target_link_libraries(pixeldb-tui
    PRIVATE ftxui::screen
    PRIVATE ftxui::dom
    PRIVATE ftxui::component
    PRIVATE pixeldb       # libpixeldb из родительского CMake
)

# ─── Результат в build/ ───
install(TARGETS pixeldb-tui
    RUNTIME DESTINATION ${CMAKE_BINARY_DIR}/../build
)
```

### 14.2 Аргументы командной строки

```bash
# Запуск с параметрами по умолчанию (127.0.0.1:5432)
./build/pixeldb-tui

# Подключиться к указанному серверу
./build/pixeldb-tui --host 192.168.1.10 --port 7777

# Показать версию
./build/pixeldb-tui --version

# Показать справку по аргументам
./build/pixeldb-tui --help
```

Парсинг аргументов — стандартный `getopt_long` или простая ручная реализация (без внешних библиотек).

## 15. UX-сценарии использования

### Сценарий 1: Первый запуск, создание структуры

```
1. Запустить: ./run.sh && ./build/pixeldb-tui
2. Увидеть Splash Screen → основной экран
3. Навигатор пуст: "No kingdoms found"
4. Нажать F9 → диалог создания БД
5. Ввести "mydb" → Enter → Create
6. Навигатор: появляется "▷ ★ mydb"
7. Навести на mydb → нажать n → диалог создания таблицы
8. Имя: "employees", добавить столбцы id INT, name VARCHAR(100), salary INT
9. Create → навигатор обновляется: "▼ ★ mydb / └ employees [0]"
10. Успех. Можно вводить запросы.
```

### Сценарий 2: Вставка и просмотр данных

```
1. Фокус на редакторе (Tab)
2. Ввести:
   INSERT INTO employees (id, name, salary)
   VALUES (1, 'Smith', 10), (2, 'Johnson', 9);
3. F5 → Results показывает: "[QUERY SUCCESS] Affected employees: 2"
4. Навигатор обновляется автоматически: employees [2]
5. Очистить редактор Ctrl+K
6. Ввести: SELECT * FROM employees;
7. F5 → Results показывает таблицу с двумя строками
8. В Results нажать ↓ чтобы выбрать строку, затем 'e'
9. Popup с детальным просмотром строки Smith
```

### Сценарий 3: Работа с историей

```
1. Выполнить несколько запросов
2. Очистить редактор Ctrl+K
3. Нажать ↑ → в редакторе появляется предыдущий запрос
4. Нажать ↑ ещё раз → ещё более ранний запрос
5. Или F2 → полноэкранная история
6. Ввести "employees" в поле Filter → остаются только запросы к этой таблице
7. Enter на нужной строке → запрос вставляется в редактор
```

### Сценарий 4: Исследование незнакомой базы данных

```
1. Подключиться к серверу с существующими данными
2. Навигатор автоматически загружает список БД и таблиц
3. Нажать Space на нужной БД → развернуть
4. Увидеть таблицы с количеством строк
5. Нажать 's' на таблице → схема в панели Results
6. Нажать 'p' → автоматически выполнить SELECT * LIMIT 10
7. В Results нажать 'f' → фильтровать строки без нового запроса
```
