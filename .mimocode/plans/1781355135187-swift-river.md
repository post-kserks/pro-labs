## C — Buffer pool интеграция (сложная)

### C1. Buffer pool для чтения страниц
- **Файл:** `server/internal/storage/page_engine.go`, `server/internal/storage/buffer_pool.go`
- **Проблема:** `getPage()` всегда читает с диска, bufPool не используется
- **Подход:** Интегрировать только для read-only операций (scanTuples, SELECT). Write path оставить прямым (dirty tracking конфликтует с текущей архитектурой)
- **Фикс:**
  1. `getPage()` → `bufPool.FetchPage(pid, hf)` с pin
  2. В `scanTuples`: unpin после обработки каждой страницы
  3. В `mutateRows`: не использовать bufPool (прямая запись через heap.WritePage)
  4. При checkpoint: bufPool.FlushAll() уже работает

## D — Архитектурные (фичи, не баги)

Эти пункты требуют серьёзного дизайна и не являются баг-фиксами. Рекомендую отложить:
- **D1.** Клиент: TLS/mTLS — нужна интеграция с OpenSSL
- **D2.** Клиент: data race в threading — зависит от FTXUI архитектуры
- **D3.** Auth token в URL для SSE — ограничение EventSource API
- **D4.** numberValue precision > 2^53 — требует нового int64 типа в JSON Value
